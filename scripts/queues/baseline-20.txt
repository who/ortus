# baseline-20.txt - Replay-queue spec for FR-023 (ralph.sh vs goal.sh comparison).
#
# Sourced by scripts/replay-queue.sh inside each freshly-cloned project. Each
# `bd create` line below seeds one issue into the project's beads DB; the
# orchestrator under test is then run with --tasks 20 to drain them.
#
# Shape mix (per AC for ortus-bn4a.1 testing instruction d):
#   - 5 chores                (trivial single-file edits or no-op closures)
#   - 10 features-with-tests  (create + verify, two-file shape)
#   - 5 bugs                  (reproduce + fix, three-file shape)
#
# All issues are designed to be tractable inside a freshly-generated project
# from `copier copy --defaults template/`. They do NOT depend on any source
# tree existing beforehand: each issue's acceptance criterion describes the
# exact files to create/modify. Priority P2 across the board so they hit
# `bd ready` in creation order; that keeps the replay deterministic.
#
# IMPORTANT: this file is sourced with `bash -e`. Any failing `bd create`
# aborts the seed and records a CRASHED row in the replay CSV. Quote
# multi-line strings with $'...' so embedded newlines survive shell parsing.
